Create your first endpoint

Logging in now only results in a Whitelabel error page. We need to serve some page or data, at some endpoint!

Fortunately, that is easy to do in Spring. Four steps are needed for that:

  1. Create a class
  2. Annotate the class with a @RestController
  3. Create a method in that class that returns some value
  4. Give the method a @GetMapping (or @Post/@Put or other mapping corresponding to a HTTP verb). Optionally: add a path.

However, this is easy to do in Spring. We just need to create a class that has some kind of @Controller annotation. As I don't want to serve a regular webpage here (we'll use React for that, later), I'll make it a @RestController.

Note: I also discuss how to activate DevTools for hot reload in IntelliJ

Create a class

I simply let IntelliJ create a class called "Controller".

public class Controller {
}

Annotate the class with a @RestController

If you don't annotate the class with @RestController, it won't create endpoints.

@RestController
public class Controller {
}

Create a method in that class that returns some value

You have to be able to give the browser some information when a request is sent. This is done via the return value of a method.

@RestController
public class Controller {
    public String showWelcome() {
        return "Hello SpringReactTutorialProject!";
    }
}

Give the method a @GetMapping (or @Post/@Put or other mapping corresponding to a HTTP verb). Optionally: add a path.

If I add @GetMapping, a request from the browser to the "empty path" (localhost:8080) will be handled by this method. I could also use @GetMapping("welcome"), in which case I would get the message if I went to localhost:8080/welcome

@RestController
public class Controller {
    @GetMapping
    public String showWelcome() {
        return "Hello SpringReactTutorialProject!";
    }
}

Extra: make Spring Boot DevTools work in IntelliJ

If, like me, you're using IntelliJ community edition, you may have noticed that DevTools, rather than automatically reloading the application on each code change, does not actually work! This can be remedied by changing some settings in IntelliJ, however, as shown on youtube. Basically:

  • Go to Settings->Compiler->Build, Execution, Deployment; activate 'Build project automatically'
  • Go to Settings->Advanced Settings; activate Allow auto-make to start

Now DevTools should work, but allow for 10 seconds or such delay after save.
