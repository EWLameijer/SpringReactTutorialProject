Step 002: Making the Spring Boot project run

The project compiles, but when I try to run it, the console prints text like:

***************************
APPLICATION FAILED TO START
***************************

Description:

Failed to configure a DataSource: 'url' attribute is not specified and no embedded datasource could be configured.

Reason: Failed to determine a suitable driver class

This means I need two things: a database, as well as a connection configured between the database and my program. Of course, to have a database, I first need to have a DBMS (Database Management System) like PostgreSQL installed on my computer. Therefore, I need three things:

  1. The DBMS: The database management system is simple to obtain; google PostgreSQL download, use the link, and download and install PostgreSQL for your operating system.
  2. The database: Simply start the pgAdmin program, login with your username and password(s) and use Servers>PostgreSQL 15>Databases, right click on Databases to create a database, and give it the appropriate name, like springreactdemo
  3. The configuration file: The configuration file is a bit more work, see below.

For the configuration I google again, for PostgreSQL and Spring, finding an example I can use.

Basically, I need to fill the src/main/resources/application.properties with something like

spring.datasource.url= jdbc:postgresql://localhost:5432/$database_name_here
spring.datasource.username= $username_here
spring.datasource.password= $password_here
spring.jpa.properties.hibernate.dialect= org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=update

Note: if IntelliJ gives a security error due to SnakeYAML, add 

<dependency>
    <groupId>org.yaml</groupId>
    <artifactId>snakeyaml</artifactId>
    <version>2.0</version>
</dependency>
to your pom.xml file.

I can now run the project, going to the port where the application is served (localhost:8080) I see a login screen, using the username "user" and the generated security password I can see in the console, I can even log in! ... to a whitelabel error page, actually, so let's fix that in the next step!
